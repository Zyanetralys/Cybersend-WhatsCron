<h2 id="que-es-esto">ВҝQuГ© es esto?</h2>

<p>CyberSend es un programador de mensajes de WhatsApp autoalojado. Escribes un mensaje, eliges una hora, y se dispara вҖ” automГЎticamente, vГӯa WhatsApp Web.</p>

<p>Se ejecuta en dos modos:</p>

<ul>
    <li><strong>HTML independiente</strong> вҖ” coloca <code>index.html</code> en cualquier lugar, incluyendo GitHub Pages. Sin servidor, sin dependencias. El navegador actГәa como programador y abre WhatsApp Web en el momento adecuado usando enlaces profundos <code>wa.me</code>.</li>
    <li><strong>Backend completo (Spring Boot)</strong> вҖ” almacenamiento persistente, API REST, actualizaciones WebSocket en tiempo real, y un despachador basado en cron adecuado. Se conecta a WhatsApp vГӯa puente de sesiГіn o la API Business oficial.</li>
</ul>

<p>La interfaz es intencionalmente exagerada: lluvia Matrix, lГӯneas de exploraciГіn CRT, tipografГӯa con efecto glitch. Porque Вҝpor quГ© no?</p>

<h2 id="integracion-whatsapp">IntegraciГіn con WhatsApp</h2>

<p>CyberSend admite tres estrategias de integraciГіn, ordenadas por complejidad:</p>

<h3>Enlaces profundos wa.me (sin configuraciГіn)</h3>

<p>La opciГіn de respaldo por defecto. Genera una URL <code>https://wa.me/{phone}?text={encoded}</code> y la abre en una nueva pestaГұa. Funciona en escritorio y mГіvil. El usuario aГәn necesita presionar Enviar manualmente вҖ” pero combinado con el programador, abre WhatsApp Web en el momento exacto.</p>

<p>Sin credenciales, sin claves API, sin gestiГіn de sesiГіn.</p>

<h3>Puente whatsapp-web.js (recomendado para automatizaciГіn completa)</h3>

<p>Ejecuta un pequeГұo acompaГұante Node.js que envuelve <a href="https://github.com/pedroslopez/whatsapp-web.js">whatsapp-web.js</a> y expone un endpoint REST local. El servicio Java llama a ese endpoint para enviar mensajes sin ninguna interacciГіn del usuario.</p>

<pre><code>cd bridge
npm install
node server.js
# Escanea el cГіdigo QR una vez вҖ” la sesiГіn se persiste en .wwebjs_auth/</code></pre>

<p>El puente expone <code>POST /send</code> con <code>{ phone, message }</code>. Edita <code>WhatsAppService.java</code> para apuntar a Г©l:</p>

<pre><code class="language-java">// WhatsAppService.java вҖ” callNodeBridge()
String url = "http://localhost:3001/send";
RestTemplate rt = new RestTemplate();
rt.postForEntity(url, Map.of("phone", phoneNumber, "message", message), String.class);</code></pre>

<h3>API Business de WhatsApp (producciГіn / alto volumen)</h3>

<p>Requiere una cuenta Meta Business verificada y un nГәmero de telГ©fono registrado. Establece tu token como variable de entorno y descomenta el bloque relevante en <code>WhatsAppService.java</code>:</p>

<pre><code>export WHATSAPP_BUSINESS_TOKEN=your_token_here
export WHATSAPP_PHONE_NUMBER_ID=your_number_id</code></pre>

<pre><code class="language-java">// WhatsAppService.java
String url = "https://graph.facebook.com/v18.0/" + phoneNumberId + "/messages";
// ... payload estГЎndar de Business API</code></pre>

<p>Se aplican lГӯmites de tasa y precios вҖ” consulta <a href="https://developers.facebook.com/docs/whatsapp/cloud-api/get-started">la documentaciГіn de Meta</a>.</p>
